Introducción a la programación en Java
Se desea crear un sistema para gestionar encuestas socioeconómicas. El sistema debe permitir registrar hasta 60 encuestados. Para cada encuestado se debe registrar:
DNI: Valor único, de tipo entero entre 10000000 y 99999999.
Nombre completo: Cadena de texto con el nombre del encuestado.
Sexo Valor entero, 1 para masculino, 2 para femenino, 3 para otro.
Edad: Valor entero entre 18 y 99.
Trabaja: Valor entero, 1 si trabaja, 2 si no trabaja.
Sueldo: Valor entero entre 0 y 500000. Si no trabaja, será 0.
Se debe mostrar un menú con las siguientes opciones:
Agregar persona
Consultar persona
Modificar persona
Eliminar persona
Listar todas las personas
Ver estadísticas
Buscar persona con mayor/menor sueldo
Salir

Métodos a utilizar: 
ingresarPersona: Mediante este método se deben ingresar los datos de una persona. 
mostrarMenuYElegirOpcion: Muestra el menú y permite elegir una opción. 
generarAccion: Según la opción elegida ejecuta la acción correspondiente. 
validarDNI: Valida que el DNI ingresado sea válido. 
buscarPersona: Busca una persona por DNI y devuelve su posición. 
consultarPersona: Muestra los datos de una persona específica. 
modificarPersona: Permite modificar los datos de una persona. 
eliminarPersona: Elimina una persona del sistema. 
listarPersonas: Muestra todas las personas registradas. 
mostrarEstadisticas: Calcula y muestra: 


Porcentaje de hombres, mujeres y otros
Porcentaje de cada género que trabaja
Sueldo promedio general y por género
Edad promedio


buscarSueldos: Encuentra personas con mayor y menor sueldo. 
obtenerPorcentaje: Calcula porcentajes para las estadísticas. 
obtenerPromedio: Calcula promedios para las estadísticas. 
ingresarEntero: Valida entradas numéricas dentro de rangos específicos.


Notas: 
Cada vez que ingrese una persona debe verificar que ésta no exista (por DNI) y que la capacidad para almacenar encuestados no haya alcanzado su máximo.
El alumno debe determinar los parámetros de entrada y salida de los métodos a desarrollar de manera que el programa sea óptimo y la mayoría o todos estos (según sea el caso) cumplan la función de poder ser reutilizables. 
Se debe corroborar siempre que el usuario ingrese un dato válido por teclado.
Los atributos no pueden ser definidos a nivel global, o a nivel clase, sino que deben ser definidos dentro de los métodos correspondientes.
Se deben utilizar las instrucciones try catch, para prevenir cualquier tipo de error que pueda surgir.
El scanner debe declararse solo en el método main y debe cerrarse como corresponde.
El alumno puede crear otras funciones si así lo considera mientras se encuentren bien implementadas.

Implementación importante para pruebas:
Para facilitar las pruebas del sistema, se debe implementar un lote de pruebas precargado. Para ello, se debe crear una constante booleana llamada MODO_PRUEBA. Si MODO_PRUEBA es true, al iniciar el programa se deben cargar automáticamente 10 ítems de ejemplo. Este lote de pruebas debe cargarse en un método separado llamado "cargarDatosPrueba". Los valores para el lote de pruebas deben generarse mediante alguna herramienta de IA como Claude, ChatGPT, Gemini, o similares para obtener datos realistas.
